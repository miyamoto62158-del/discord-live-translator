# Oracle Cloud 無料枠によるハイブリッド環境のデプロイ状況と引き継ぎガイド 🌐

このドキュメントは、Discord Live Translator をクラウド（Oracle Cloud）とローカルGPUの「ハイブリッド構成」でデプロイするための**現在の構築進捗状況**と、共同開発者向けの**今後の引き継ぎ手順**をまとめたものです。

---

## 1. ハイブリッド構成の全体像 🏗️

今回のハイブリッド化により、以下の役割分担で24時間いつでもBotを稼働させつつ、重いGPU処理はローカルで分担する仕組みを実現しています。

*   **クラウド（Oracle Cloud VMインスタンス）**:
    *   Discord Botの常時稼働・ログイン維持
    *   Webダッシュボードの配信（ブラウザで閲覧可能）
    *   ローカルクライアントと連携するWebSocketサーバーの稼働
*   **ローカルPC（GPU搭載環境）**:
    *   ユーザーが `/join` でBotをボイスチャンネルに入れた際に、WebSocket経由で音声ストリームを受信
    *   ローカルのNVIDIA GPUを利用した Qwen-ASR（音声認識文字起こし）
    *   DeepLによる翻訳結果をWebSocket経由でクラウドに送信

---

## 2. 現在のデプロイ進捗状況 📍

Oracle Cloud上でのインフラ構築は **「ほぼ完了」** していますが、最後のVMインスタンス作成時に無料枠のサーバー空き不足により、設定を**保留（スタック保存）**している状態です。

### ✅ 構築完了しているネットワーク（VCN）設定
以下のインフラはすでにOracle Cloud上に構築済みです。これらを新しく作り直す必要はありません。

1.  **VCN（仮想クラウドネットワーク）**: `live-translator-vcn`
2.  **パブリック・サブネット**: `public-subnet` (Regional, 10.0.0.0/24, DNS解決ON)
3.  **インターネット・ゲートウェイ ＆ ルート・テーブル**: 設定済み（サブネットのパブリックIPトグルをONにできる構成）
4.  **セキュリティ・リスト**: VCN用のデフォルトのリストが割り当て済み

### ⚠️ 現在の保留ステータス（VMインスタンス作成）
*   **エラー内容**: `Out of capacity for shape VM.Standard.A1.Flex in availability domain AD-1...`
*   **原因**: バタム（AP-BATAM-1 / インドネシア）リージョンにおいて、人気の高い無料Armインスタンス（A1.Flex）の物理的な在庫が一時的に不足しているため。
*   **現在の状態**: インスタンスの設定内容（OS: Ubuntu 24.04, SSH公開鍵, VCN接続など）を **スタック名: `live-translator`** として完全に保存（保留）しています。

## 3. 共同開発者が行うべき次のステップ 🛠️

このプロジェクトを引き継ぐ場合、以下の手順に沿って環境の構築を完了させてください。

### アプローチA：保留中のスタックから再実行してみる（無料のまま待つ）
時間帯（平日の早朝など）によって空きが出る可能性があるため、定期的にスタックから再試行（Apply）を実行します。

1.  Oracle Cloud 管理画面の左上メニュー ☰ から **「Developer Services」** → **「Stacks」** に進みます。
2.  保存されている **`live-translator`** というスタックをクリックします。
3.  画面上部にある **「Apply (適用)」** ボタンをクリックし、確認画面でそのまま「Apply」を実行します。
4.  **Succeeded（成功）** になればVMインスタンスのデプロイが完了します！(パブリックIPが発行されます)

### アプローチB：アカウントを有料（Pay-As-You-Go）にアップグレードする（即時完了・推奨）
空き待ちをせずに、今すぐデプロイを完了させる最も確実な方法です。

1.  Oracle Cloud アカウントを「Pay-As-You-Go（従量課金制）」にアップグレードします。
2.  有料アカウントになると空き枠の優先度が上がるため、上記スタックから **「Apply」** を実行すると、即座にVMインスタンスが作成されます。
3.  **【超重要】** 有料アカウントになっても、今回設定している **`VM.Standard.A1.Flex` (1 OCPU / 6 GBメモリ / 約47 GBブートボリューム) の範囲内であれば、Always Free（無料枠）として認識され、月額料金は 0円（完全無料）のまま維持されます。** (無料枠を超えない限り請求は発生しません)

## 4. インスタンス作成完了後のデプロイ作業 🚀

VMインスタンスが無事に作成された（または作成を試みる）後の手順は、プロジェクト内の以下のセットアップガイドに詳細が記述されています。

*   **詳細手順書**: [`oracle_cloud_setup_guide.md`](./oracle_cloud_setup_guide.md)

### 概要ステップ：
1.  発行された **パブリックIP** を使用して、以下のコマンドでSSH接続します。
    ```bash
    ssh -i C:\Users\miyam\.ssh\oracle_key ubuntu@[パブリックIP]
    ```
2.  ガイドに沿って、Node.js 20、Git のインストール、およびリポジトリのクローンを行います。
3.  クラウド上のBot配下の `.env` を設定します（Discordトークン、DeepLキー等の設定）。
4.  **重要**: ダッシュボードおよびWebSocket通信のために、**ポート `3000`** を開放する必要があります。
    *   OS側のファイアウォール（iptables）の開放コマンドを実行。
    *   Oracle Cloud管理画面で、セキュリティ・リストにポート `3000` を追加（Source: `0.0.0.0/0`）。
5.  `pm2` を使用して、Bot ＆ WebSocketサーバーを24時間自動起動化します。
    ```bash
    pm2 start index.js --name live-translator
    ```
